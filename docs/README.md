# Docs

Project documentation and artifacts for the Revenue Command Center (Pattern 4). The app itself lives in [`../pattern4-agent-portal/`](../pattern4-agent-portal/).

| Folder | Contents |
| --- | --- |
| [`demo/`](demo/) | Demo runbook, booth vignettes, solution summary, and deck outlines — the narrative deliverables. |
| [`planning/`](planning/) | Shaping notes, spike findings, and redesign plans captured while building. |
| [`reference/`](reference/) | Manifests, model/endpoint reports, synthetic-data specs, AI Gateway/OBO notes, and the style guide. |
| [`data/`](data/) | Dataset inventories, federation/migration probes, Genie space captures, and UC samples (generated snapshots). |
| [`archive/`](archive/) | Superseded scratch artifacts kept for history (early screenshots, throwaway API captures, working notes). |

The live scope + sprint tracker stays at the repo root: `pattern-4-agent-to-agent-automation-scope-build-plan.md` (source of truth), with `pattern-4-plan-data.js` / `pattern-4-project-manager.html` as the browser-openable tracker.
